# IONe v0.3 design note — provenance footers + semantic-layer-first

Source: https://claude.com/blog/how-anthropic-enables-self-service-data-analytics-with-claude
(Ryan, 2026-06-07. Anthropic's internal agentic analytics stack: 21%→95% eval accuracy.)

This is validation, not new direction — Anthropic's "adversarial-review sub-agent + provenance"
maps almost 1:1 onto IONe's generator↔critic→routing loop + approval gate. Two borrowable
specifics for v0.3, both leveraging machinery we already have.

## 1. Provenance footers on rendered answers

Anthropic stamps every answer with source tier + freshness; it is their cheapest trust lever.
IONe already has event lineage — surface it. Every generator output (feed/notification/draft)
carries a footer: originating connector, source tier (canonical vs. raw), and event freshness
(max event timestamp + age). Pairs naturally with the planned v0.3 pgvector event search —
retrieved events already carry the metadata the footer needs.

Acceptance: a rendered answer shows e.g. `source: <connector> · tier: canonical · fresh: 6m ago`,
populated from the event records that fed the generator, not re-derived.

## 2. Semantic-layer-first as a hard skill rule

The 21%→95% lesson: the bottleneck was *structure* (question→entity mapping), not information
access — giving the agent thousands of historical queries moved accuracy <1pt. The fix is a
semantic layer the agent is **structurally required** to consult before free-form querying.

For IONe's federation fabric: when a connector exposes a data app over MCP, it should expose
**compiled metric definitions**, and the generator skill must be required (by instruction) to
resolve against those metrics before issuing raw queries. This is the federation-layer analog of
their "consult the semantic layer first" rule. Treat connector metric definitions as a first-
class MCP capability, not an afterthought.

Acceptance: generator skill instruction mandates metric resolution first; a connector with no
compiled metrics degrades gracefully (free-form) but logs the gap.

## Note for CAGE (separate repo)

Anthropic's finding that ~90% of data-model PRs ship a paired skill/doc change — enforced by a
CI hook — is external validation of CAGE's reason to exist (durable spec + native realization +
golden fixtures, kept in sync mechanically; ADR-0002 already emits SKILL.md). Use as positioning
evidence ("the maintenance-as-engineering problem solved ad hoc with CI hooks is what a module
registry solves structurally"), not a code change.
